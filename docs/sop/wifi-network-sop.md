# SOP 06: Wi-Fi / Network Troubleshooting

**Document Title:** SOP – Wi-Fi / Network Troubleshooting  
**Document Code:** SOP-NET-006  
**Version:** 1.0  
**Owner:** IT Network Team  
**Last Updated:** 2026-04-14  

---

## 1. Purpose
To resolve common network and Wi-Fi connectivity issues affecting user access to the internet or internal systems.

---

## 2. Scope
Applies to users experiencing:
- No internet connection  
- Wi-Fi not connecting  
- Limited access  
- Network drops  
- Slow internet  

---

## 3. Requirements
- Active router/network connection  
- User device network enabled  
- Admin access (if required)  

---

## 4. Common Symptoms
- “No internet access”  
- Wi-Fi not visible  
- Connected but no internet  
- Slow browsing  
- Network disconnecting frequently  

---

## 5. Procedure

### 5.1 Basic Checks
- Confirm Wi-Fi is turned ON  
- Check airplane mode is OFF  
- Restart device  

---

### 5.2 Router Check
- Restart router (power off for 10–15 seconds)  
- Wait for full reboot  
- Check indicator lights (Internet/Wi-Fi)

---

### 5.3 Reconnect Network
- Forget Wi-Fi network  
- Reconnect using password  
- Verify connection stability  

---

### 5.4 IP Configuration Reset
- Open Command Prompt  
- Run:
  - ipconfig /release  
  - ipconfig /renew  
  - ipconfig /flushdns  

---

### 5.5 Advanced Checks
- Test with another device  
- Check network cable (if Ethernet)  
- Disable VPN if active  
- Update network drivers  

---

## 6. Escalation
Escalate to Network Team if:
- No internet after all steps  
- Router failure suspected  
- Network outage confirmed  
- Multiple users affected  

---

## 7. Ticket Classification
- Category: Network  
- Subcategory: Wi-Fi / Connectivity  
- Priority: High (if user cannot work)

---

## 8. Technician Notes
Network issue diagnosed and resolved or escalated. Connectivity restored or ticket forwarded to Network team.
