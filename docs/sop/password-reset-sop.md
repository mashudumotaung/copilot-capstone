# SOP 01: Password Reset & Account Unlock

**Document Title:** SOP – Password Reset & Account Unlock  
**Document Code:** SOP-ACC-001  
**Version:** 1.0  
**Owner:** IT Service Desk  
**Last Updated:** 2026-04-14  

---

## 1. Purpose
To provide a standard procedure for resetting user passwords and unlocking accounts to restore access to company systems.

---

## 2. Scope
Applies to all employees experiencing:
- Forgotten password  
- Password expired  
- Account locked  
- Login failure due to incorrect credentials  

---

## 3. Requirements
- User must confirm identity (employee number or ID verification)  
- Admin access to Active Directory / Azure AD (if applicable)

---

## 4. Symptoms
- “Your password has expired”  
- “Account locked”  
- “Incorrect username or password”  
- User cannot log in to PC/email  

---

## 5. Procedure

### 5.1 Verify User Identity
- Ask for full name  
- Ask for employee number  
- Confirm department and manager  
- Confirm email or phone number  

---

### 5.2 Reset Password
- Open Active Directory / Azure Admin Center  
- Locate user account  
- Click “Reset Password”  
- Generate temporary password  
- Enable “User must change password at next login”  
- Share password securely  

---

### 5.3 Unlock Account
- Locate user account  
- Click “Unlock Account”  
- Test login access  

---

### 5.4 User Guidance
- Log in using temporary password  
- Create new secure password:
  - 8–12 characters minimum  
  - Uppercase + lowercase + number + symbol  
- Confirm login success  

---

## 6. Escalation Conditions
Escalate to IAM/Identity team if:
- Account is disabled  
- Repeated lockouts occur  
- MFA reset required  
- Security breach suspected  

---

## 7. Ticket Classification
- Category: Access / Account  
- Subcategory: Password Reset  
- Priority: Medium (High if urgent access needed)  

---

## 8. Technician Notes
Password reset completed and account unlocked. User successfully logged in after reset.
